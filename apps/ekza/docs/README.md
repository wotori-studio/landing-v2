# Ekza Documentation

This directory contains documentation for the Ekza project.

## Structure

All project documentation should be placed in this `docs/` directory to keep the project root clean and organized.

## Available Documentation

- **NEWSLETTER_SETUP.md** - Complete setup guide for newsletter subscription functionality
- **QUICK_START_NEWSLETTER.md** - Quick start guide for newsletter setup
- **analytics-setup.md** - Privacy-friendly analytics system setup and usage guide

## Adding New Documentation

When creating new documentation files:

1. Place them in this `apps/ekza/docs/` directory
2. Use descriptive, kebab-case filenames (e.g., `feature-setup.md`)
3. Write documentation in English (this is an international repository)
4. Include a brief description in this README if the documentation is important

## For AI Assistants

**Important:** When creating documentation files for this project, always place them in `apps/ekza/docs/` directory, not in the project root. This keeps the project structure clean and organized.
